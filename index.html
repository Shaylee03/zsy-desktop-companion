Project Overview

一个围绕“主动式 AI 介入边界”的产品原型

项目不是做一个普通桌面提醒工具，而是把桌面状态理解、可打扰性判断、机器人化反馈和个性化记忆组织成完整闭环。

项目目标 解决主动 AI 有用但容易打扰的问题

让机器人先理解用户当前状态，再决定是否介入、用什么强度表达、是否延后或静默。

我的角色 AI 产品设计、交互策略与原型呈现

拆解使用场景，定义状态判断规则、四级反馈策略、记忆机制和验证口径。

核心产出 状态感知、介入策略、记忆机制和机器人反馈

Problem

主动式 AI 的难点不是“会不会说”，而是“该不该介入”

传统提醒按时间触发，普通助手依赖用户主动提问。桌面机器人一旦变得更主动，就必须先回答：用户现在是否适合被打扰？这次介入应该静默、轻动作、字幕，还是语音？

普通提醒工具按固定时间或事件触发，较少理解会议、专注、离席返回等桌面状态。
本项目方案先判断状态和置信度，再选择最低可行反馈强度，并把用户响应回流到记忆策略。

User Scenario

从高打扰风险场景拆出主动交互规则

场景不是为了堆功能，而是用于定义机器人何时沉默、何时弱提示、何时给字幕，以及什么情况下才允许语音。

Interaction Strategy

同一次主动机会，会因为用户状态而改变存在感

策略层始终选择最低可行打扰强度，让机器人先学会沉默，再学会表达。

quiet会议、深度专注、不确定状态：静默记录，不主动出现。
subtle低风险提示：表情、灯效或轻动作，弱存在感。
subtitle离席返回、任务恢复：用短字幕恢复上下文。
speak空闲、高置信、重要事项：才允许语音播报。

会议中收到低优先级事项

状态会议置信度高。

策略语音阻止，进入 quiet 或 subtitle。

Memory Mechanism

记忆不是越多越好，而是持续修正主动边界

系统不默认保存所有行为，而是根据任务相关性、重复性、用户反馈和时间衰减，决定信息是否进入长期记忆。

记录 -> 总结 -> 合并 -> 调用 -> 更新 -> 降权 基于长期反馈更新用户偏好权重和主动介入边界

Reflection / Value

它的价值不在于“提醒用户做事”，而在于设计主动式 AI 的介入边界

项目从产品问题、交互机制、系统实现到验证记录形成闭环，重点呈现主动式 AI 在桌面工作流中的介入边界设计。

AI 产品判断把模型能力放进状态规则、置信度、偏好和安全边界中，而不是让模型直接决定高打扰行为。
HRI 交互策略把机器人表达拆成静默、轻动作、字幕和语音四级，控制存在感和打扰感。
具身智能表达机器人不是通知容器，而是根据桌面状态选择姿态、文字和声音强度的桌面智能体。
工程与验证闭环本地采集、云端策略、记忆机制、规则回放和页面证据保持一致。
后续可补充材料 真实演示录屏、更多桌面场景样本、记忆编辑界面、长期使用反馈。
