Project Overview

一个把桌面状态、主动交互和机器人表达连起来的产品原型

这个项目关注一个放在电脑旁边的机器人，如何理解用户当下的学习 / 工作节奏，并在合适的时候用合适的表达方式提供陪伴和支持。

产品定位 理解桌面节奏的主动式 AI 陪伴机器人

面向长时间学习与办公的人群，在保持安静共在的基础上，根据桌面状态选择合适的表达时机和强度。

我的角色 产品定义、交互策略、前后端原型与硬件联调

独立完成产品规划、Windows 感知应用、云端策略与记忆接口，并把现有桌面机器人接入完整链路。

产品形态 Windows 应用 + 云端智能 + 实体机器人

本地端负责感知与隐私控制，云端负责策略和记忆，机器人负责表情、字幕、动作与语音反馈。

Problem

电脑前的人需要一个懂节奏的桌面伙伴

研究生和办公人员经常长时间坐在电脑前：资料、论文、代码、会议和消息不断切换。体验痛点集中在低能量、任务漂移、孤立工作，以及工具难以理解当下节奏带来的状态断裂。

普通 AI 工具等用户主动打开、主动提问，无法感知当前桌面节奏，也很难在用户低能量或任务漂移时自然出现。
本项目方案把电脑使用状态、机器人状态和用户反馈放进同一条链路，让机器人通过表情、字幕、轻动作或语音成为低压力桌面伙伴。

System Logic

从桌面状态到机器人反馈，再回到记忆更新

桌面状态是实现主动陪伴的关键路径。Windows 应用保留本地节奏轨迹和隐私控制，云端连接策略与记忆，机器人把结果转成具身反馈。

01桌面信号前台窗口、键鼠、AFK、系统音频、应用使用时长
02状态摘要学习、写作、会议、任务中断、工作状态分散、返回电脑
03主动机会是否需要陪伴、任务线索恢复、低压力状态反馈或静默记录
04反馈策略根据状态风险、偏好、冷却和机器人可用性选择强度
05机器人表达表情、轻动作、底部字幕、语音播报
06记忆更新接受、忽略、关闭、延后、纠正和场景偏好

Interaction Strategy

低打扰是主动陪伴必须遵守的边界

机器人越主动，越需要克制。策略层通过规则、置信度、用户偏好和硬件状态选择最低可行反馈强度，让表达保持可解释和可控。

quiet会议、深度专注、信号不足：只记录，不主动出现。
subtle低风险陪伴：表情、灯效或轻动作，弱存在感。
subtitle任务恢复、低压力说明：用短字幕表达，不直接语音打断。
speak空闲、高置信、重要事项：才允许语音播报。

Memory Mechanism

记忆会随反馈更新，也会在证据不足时降权

这里的记忆不是对用户做固定画像，而是保存可追溯的偏好与节奏假设，并根据新证据持续校正。

候选形成 -> 置信度评估 -> 调用 -> 反馈校正 -> 衰减或删除 系统只在内容相关且置信度足够时调用记忆；用户纠正、长期未验证或出现相反证据时，记忆会更新或降权。
Evidence

从重复证据形成候选

桌面摘要和交互反馈先形成候选，不把单次行为直接当作长期偏好。

Update

显式与隐式反馈共同校正

纠正、删除和确认是强信号；接受、忽略和延后只用于调整置信度与表达策略。

Control

用户可以查看和修改

偏好与节奏记忆保留来源、置信度和更新时间，支持确认、纠正、删除与时间衰减。
