# 产品案例

## 项目背景

研究生和办公人员经常长时间坐在电脑前：写论文、查资料、改代码、开会、回消息、切换任务。很多工具可以提高效率，但很少有工具能像桌面伙伴一样理解用户正在经历的学习 / 工作节奏。

这个项目的核心问题是：

> 桌面 AI 机器人如何根据电脑使用状态和用户反馈，适时提供低压力陪伴，并随着使用逐步理解用户习惯？

因此，“低打扰”被定义为主动陪伴必须遵守的边界。项目重点关注主动陪伴、桌面状态感知、机器人化反馈，以及基于反馈持续校正的记忆。

## 目标用户

| 用户类型 | 典型状态 | 产品价值 |
| --- | --- | --- |
| 学生 / 研究生 | 长时间学习、写作、查资料，容易低能量或任务漂移 | 提供低压力陪伴、任务聚焦和学习节奏支持 |
| 办公 / 知识工作者 | 会议、消息和任务切换频繁 | 根据状态选择静默、字幕、动作或语音，减少不合时宜的打断 |
| AI 工具高频用户 | 希望 AI 更主动，但不想被频繁干扰 | 把主动 AI 变成可解释、可降级、可学习的桌面伙伴 |

## 核心场景

1. 长时间学习低能量：输入下降、窗口切换增多或停留无推进时，机器人用轻动作或短字幕提供节奏支持。
2. 任务漂移恢复：用户离开原任务后，系统在合适时机用字幕恢复任务线索。
3. 会议勿扰：会议中禁止普通语音反馈，必要信息降级为静默或短字幕。
4. 深度专注保护：生产力应用持续前台且输入稳定时，普通主动机会延后。
5. 返回电脑：AFK 后恢复输入时，先用短字幕恢复上下文，不直接语音打断。

离席返回是“桌面伙伴如何帮助恢复上下文”的一个子场景。

## 产品机制

```text
桌面信号
  -> 桌面状态摘要
  -> 主动机会判断
  -> 反馈强度选择
  -> 机器人表达
  -> 用户响应
  -> 记忆更新
```

关键设计取舍：

- 优先使用前台窗口、键鼠、AFK、系统音频和 ActivityWatch 等非摄像头桌面信号。
- 本地感知端负责生成桌面状态摘要；后端 Agent 能力用于理解状态、生成表达和总结反馈，触发强度由规则、置信度、偏好和硬件状态共同约束。
- 机器人反馈通过表情、动作、底部字幕和语音形成多级表达。
- 记忆记录的是任务上下文、偏好、节奏和反馈，不追求保存所有用户行为。

## 当前产出

- 本地桌面感知端与云端 `/desktop-context` 链路。
- 云端 proactive brain、状态归一化、主动机会识别、反馈强度策略。
- 任务上下文、场景偏好、节奏摘要和互动反馈等记忆结构。
- 基于 ESP32-S3N16R8-EMOJI 开源硬件 / 固件框架的机器人联调。
