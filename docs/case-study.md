# 产品案例

## 项目背景

桌面学习、办公和远程协作用户长期处在会议、专注、阅读、消息切换和离席恢复之间。传统提醒工具只会按时间触发，普通语音助手又主要依赖用户主动唤醒。直接让 AI 更主动，会带来新的问题：它可能在会议、专注和低能量状态里变成新的干扰源。

## 核心问题

本项目不把问题定义为“如何提醒用户”，而是定义为：

> 主动式 AI 如何判断什么时候该介入、什么时候该延后、什么时候该静默、什么时候才允许语音？

因此产品重点是“主动介入边界”，不是“功能越多越好”。

## 目标用户

| 用户类型 | 核心痛点 | 产品价值 |
| --- | --- | --- |
| 学生 / 研究生 | 长时间学习、容易忘记任务、休息节奏不稳定 | 帮助恢复任务线索，减少机械打断 |
| 远程办公 / 知识工作者 | 会议多、任务切换频繁、消息容易打断专注 | 判断会议和专注状态，减少不合时宜打扰 |
| 智能工具高频用户 | 希望桌面助手更主动，但不想被频繁干扰 | 提供个性化主动助手体验 |

## 不优先服务

- 泛娱乐陪聊用户。
- 完全依赖语音交互的家庭场景。
- 高隐私敏感但不愿授权任何桌面信号的用户。
- 需要复杂企业自动化执行的场景。

## 高价值场景

1. 会议勿扰：会议中默认禁止普通主动语音，低优先级事项静默记录。
2. 深度专注保护：专注中延后普通主动机会，必要介入降级为字幕或轻动作。
3. 离席返回摘要：用户回来后先用字幕恢复上下文，不立刻语音打断。
4. 连续工作状态干预：不机械按时间打断，而是结合状态选择合适时机。
5. 任务遗忘恢复：频繁切换后，在可打扰状态下恢复原任务线索。
6. 低能量状态支持：不做心理诊断，只做低压力任务拆分或休息建议。

## 产品取舍

- 不使用摄像头作为默认输入，优先使用低隐私侵入的桌面信号。
- 语音不是默认表达，机器人主要通过静默、表情、动作和字幕降低打扰。
- 大模型不直接控制高打扰行为，触发和强度由规则、置信度、偏好和安全边界控制。
- 记忆不追求越多越好，必须可解释、可更新、可删除、可降权。

## 当前产出

- 本地桌面采集端与云端 `/desktop-context` 链路。
- 桌面状态识别与 rhythm event 原型。
- `quiet / subtle / subtitle / speak` 四级反馈策略。
- 会议、忙碌、硬件离线、冷却、预算、偏好降级逻辑。
- 分层记忆、主动反馈记录和自演化策略补丁雏形。
- 公开作品集、本地采集端链路、验证记录和隐私边界说明。
