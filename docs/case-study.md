# 产品案例

## 项目背景

桌面学习和办公用户长时间处在会议、专注、阅读、消息切换和离席恢复等状态中。传统语音助手只在用户唤醒后工作，而简单增加主动提醒又会破坏工作节奏。

## 核心问题

项目关注的不是“机器人能不能主动说话”，而是“什么时候的主动介入是帮助，什么时候会变成打扰”。

## 产品取舍

- 不依赖摄像头视觉监控，优先使用桌面状态信号。
- 不将所有主动交互都转成语音，优先使用静默、表情动作和字幕。
- 不读取微信正文，只使用进程、前台窗口和输入活跃状态。
- 信号不可用或硬件离线时，只记录，不猜测、不假装执行。

## MVP 优先级

1. 会议勿扰。
2. 专注保护。
3. 离席返回恢复。

阅读思考、消息打断、连续工作和疲劳作为扩展场景。

## 产出物

- 本地桌面状态采集端。
- 云端对话编排和主动交互决策。
- HRI 交互策略矩阵。
- 分层记忆与 Hermes 反思机制原型。
- Agent 可观测和策略配置后台。
- 路由延迟基线和产品验证计划。
