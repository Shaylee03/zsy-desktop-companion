# 指标与验证

## 验证结论边界

当前已经建立的是“路由延迟和策略观测口径”，不是大样本体验结论。误打扰率、提醒接受率、忽略率、降级率和记忆命中率仍需要通过访谈、自测和人工标注回放补齐。

## 当前真实基线

| 路由 | 样本 | LLM 平均 | LLM P50 | LLM P95 |
| --- | ---: | ---: | ---: | ---: |
| fast_chat | 3 | 932.0ms | 884ms | 1073.9ms |
| deep_agent | 66 | 6129.1ms | 5527.5ms | 11818.0ms |

数据截止于 2026-06-18，来自最近 75 条对话路由日志。fast 样本仅 3 条，因此只用于说明“已建立观测口径并获得初步趋势”，不作显著性结论。

## 已建立口径

- fast/deep 路由比例。
- LLM、TTS 和端到端延迟。
- fallback 原因。
- 记忆写入和跳过原因。
- 主动交互压制原因。

## 待完成验证

- 3–5 名目标用户访谈。
- 连续 3 天真实桌面使用记录。
- 误打扰率、接受率、忽略率和降级率。
- 会议、专注、阅读、离席状态人工标注回放。
- 硬件在线时表情、动作、字幕和语音成功率。

详细访谈和自测模板见 [访谈与 3 天自测计划](research-and-self-test.md)。

## 指标字典

| 指标 | 用途 | 是否已有结论 | 面试中推荐说法 |
| --- | --- | --- | --- |
| 路由延迟 | 判断 fast/deep 路由的响应成本 | 有小样本基线 | 已建立延迟观测口径，样本仍需扩大 |
| 误打扰率 | 判断主动介入是否打断用户 | 无 | 已规划口径，待 3 天自测和标注回放 |
| 提醒接受率 | 判断提醒是否有实际帮助 | 无 | 已规划口径，待目标用户验证 |
| 策略降级率 | 判断风控策略是否生效 | 无 | 已定义压制和降级原因，待日志统计 |
| 记忆命中率 | 判断偏好记忆是否影响决策 | 无 | 记忆效果仍在原型阶段，不写成果 |
| 硬件下发成功率 | 判断表达层是否稳定 | 无 | 需要硬件在线测试后再写 |
