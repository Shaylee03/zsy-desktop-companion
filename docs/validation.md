# 指标与验证

## 验证结论边界

当前已经建立的是“路由延迟、策略观测和指标口径”，不是大样本体验结论。误打扰率、提醒接受率、忽略率、策略降级率、任务恢复率和记忆命中率仍需要通过访谈、自测和人工标注回放补齐。

## 当前真实基线

截至公开材料整理时，已有：

- 75 条路由日志样本。
- fast/deep 路由延迟小样本统计。
- 主动策略状态、冷却、预算、压制原因和降级原因字段。
- 分层记忆写入与跳过原因。
- 本地 Agent 和云端 `/desktop-context/status` 可观测链路。

这些只能证明系统具备观测能力和初步工程链路，不能证明体验已经显著优化。

## 低打扰指标

| 指标 | 计算方式 | 当前状态 | 面试推荐说法 |
| --- | --- | --- | --- |
| 误打扰率 | 用户标记为不该提醒的主动输出数 / 主动输出总数 | 待标注 | 已规划口径，待 3 天自测和回放标注 |
| 语音误触发率 | 会议/专注中错误使用语音次数 / 高敏状态总次数 | 待标注 | 已设计会议禁语音和专注降级边界 |
| 策略降级率 | 因会议、冷却、信号不足、离线、偏好降级次数 / 候选机会总数 | 字段已具备 | 可观测降级原因，结论待样本 |
| 用户关闭率 | 用户手动关闭提醒次数 / 主动输出总数 | 待标注 | 作为负反馈进入偏好更新 |
| 忽略率 | 无响应提醒次数 / 主动输出总数 | 待标注 | 用于调节频率和强度 |

## 系统性能指标

| 指标 | 用途 | 当前状态 |
| --- | --- | --- |
| 端到端响应延迟 | 从状态变化到机器人反馈的时间 | 需结合本地 Agent、后端和硬件日志 |
| 状态识别置信度 | 判断当前状态可靠性 | 已有状态 confidence 雏形 |
| 本地处理占比 | 评估隐私友好程度 | 本地 Agent 负责桌面采集与摘要 |

## 下一步验证

1. 采集会议、专注、离席、返回、任务切换、疲劳各不少于 10 条人工标注样本。
2. 做 3 天自测，记录每次主动输出的场景、输出等级、用户反馈、是否误打扰、是否采纳。
3. 访谈 3 到 5 名目标用户，验证场景优先级和反馈强度偏好。
4. 回放标注日志，计算误打扰率、接受率、降级率、任务恢复率和记忆命中率。
