# 访谈与 3 天自测计划

这份文档用于补齐用户研究和体验验证证据。当前状态是“验证计划与记录模板已建立”，不是“已完成用户验证”。在没有真实记录前，简历不应写误打扰率下降或接受率提升。

## 验证目标

1. 判断桌面学习/办公用户是否真的需要低打扰主动介入。
2. 验证会议、专注、离席返回是否是优先级最高的 MVP 场景。
3. 观察四级反馈中哪些表达会被认为有帮助，哪些会被认为打扰。
4. 建立误打扰率、接受率、忽略率、降级率、端到端延迟等指标口径。

## 目标访谈对象

| 类型 | 数量建议 | 筛选条件 |
| --- | ---: | --- |
| 学习用户 | 2 | 每天连续学习或写作超过 3 小时 |
| 办公用户 | 2 | 高频会议、文档、消息切换 |
| AI 工具重度用户 | 1 | 经常使用 AI 助手或桌面效率工具 |

## 访谈提纲

| 问题 | 想验证什么 |
| --- | --- |
| 你在哪些桌面场景下最不希望被提醒打断？ | 高风险场景 |
| 哪些提醒如果来得及时，会真的帮到你？ | 高价值主动机会 |
| 你能接受机器人用哪些方式提醒你？ | 反馈强度偏好 |
| 你是否愿意让系统读取窗口、键鼠、AFK、音频状态？ | 隐私边界 |
| 如果机器人判断错了，你希望如何纠正它？ | 反馈回流设计 |
| 你更介意“错过提醒”还是“错误打扰”？ | 策略倾向 |

## 访谈记录模板

| 字段 | 记录方式 |
| --- | --- |
| 受访者编号 | P01、P02，不记录真实姓名 |
| 用户类型 | 学习、办公、AI 工具重度用户 |
| 高频场景 | 会议、专注、阅读、消息处理、离席返回 |
| 可接受信号 | 窗口、键鼠、AFK、音频状态 |
| 不可接受信号 | 摄像头、正文内容、屏幕截图等 |
| 高价值提醒 | 用户主动提到的可接受提醒 |
| 高打扰行为 | 用户明确反感的提醒方式 |
| 对四级反馈的排序 | quiet/subtle/subtitle/speak 的偏好 |
| 对原型的主要质疑 | 隐私、准确性、打扰、价值等 |

## 3 天自测方案

| 天数 | 目标 | 记录重点 |
| --- | --- | --- |
| Day 1 | 验证会议勿扰与专注保护 | 会议中是否被压制、专注时是否误触发 |
| Day 2 | 验证离席返回与阅读思考 | AFK 恢复是否识别、阅读是否过度提醒 |
| Day 3 | 验证降级策略和冷却预算 | 信号不足、硬件离线、冷却中是否低风险处理 |

## 指标计算口径

| 指标 | 计算方式 | 当前状态 |
| --- | --- | --- |
| 误打扰率 | 被用户标记为打扰的主动输出数 / 主动输出总数 | 待真实自测 |
| 提醒接受率 | 用户采纳或正反馈的提醒数 / 主动输出总数 | 待真实自测 |
| 忽略率 | 无响应或主动关闭的提醒数 / 主动输出总数 | 待真实自测 |
| 策略降级率 | 因会议、冷却、信号不足、硬件离线而降级的次数 / 候选机会总数 | 待真实自测 |
| 记忆命中率 | 触发用户偏好规则的次数 / 可使用记忆的决策次数 | 待真实自测 |
| 端到端延迟 | 从候选机会生成到机器人完成表达的耗时 | 已有路由延迟小样本，端到端仍待补 |

## 访谈完成后的输出格式

完成访谈后，建议只公开聚合结论：

- 受访者数量和类型。
- 高频痛点排名。
- 用户对四级反馈的偏好。
- 用户对隐私边界的接受度。
- 对 MVP 场景优先级的修正。

不要公开原始聊天记录、个人身份、桌面截图或隐私上下文。
