# 研究方法与体验评估口径

这份文档记录项目使用的研究方法和体验评估口径。当前公开页面展示的是已经完成的自动化验证、页面可用性检查、下载包检查和云端烟测；长期多人使用数据属于产品化阶段继续积累的内容。

## 研究目标

1. 判断桌面学习/办公用户在什么场景下最不希望被打断。
2. 验证会议、专注、离席返回是否应该成为 MVP 的核心场景。
3. 判断 quiet / subtle / subtitle / speak 四级反馈在不同状态下是否合理。
4. 建立误打扰率、接受率、忽略率、降级率、端到端延迟等指标口径。

## 用户类型

| 类型 | 关注点 |
| --- | --- |
| 学习用户 | 长时间学习、写作、资料阅读和任务恢复 |
| 办公用户 | 会议、文档、消息切换和远程协作 |
| AI 工具高频用户 | 希望 AI 更主动，但不希望被频繁打断 |

## 访谈问题池

| 问题 | 想验证什么 |
| --- | --- |
| 哪些桌面场景下最不希望被提醒打断？ | 高风险打扰场景 |
| 哪些提醒如果来得及时，会真的帮到你？ | 高价值主动机会 |
| 你能接受机器人用哪些方式提醒你？ | 反馈强度偏好 |
| 你是否愿意授权窗口、键鼠、AFK、音频状态？ | 隐私边界 |
| 如果机器人判断错了，你希望如何纠正它？ | 反馈回流设计 |
| 你更介意错过提醒，还是错误打扰？ | 策略倾向 |

## 原型自测口径

| 场景 | 观察重点 |
| --- | --- |
| 会议勿扰 | 是否压制语音，是否延后低优先级事项 |
| 深度专注 | 是否减少主动出现，是否在结束后总结 |
| 离席返回 | 是否先字幕摘要，是否避免突然说话 |
| 任务恢复 | 是否提供具体上下文线索，而不是空泛催促 |
| 疲劳提醒 | 是否根据状态选择时机，而不是机械定时 |
| 低能量支持 | 是否避免情绪诊断式表达 |

## 指标口径

| 指标 | 计算方式 |
| --- | --- |
| 误打扰率 | 被用户标记为打扰的主动输出数 / 主动输出总数 |
| 提醒接受率 | 用户采纳或正反馈的提醒数 / 主动输出总数 |
| 忽略率 | 无响应或主动关闭的提醒数 / 主动输出总数 |
| 策略降级率 | 因会议、冷却、信号不足、硬件离线而降级的次数 / 候选机会总数 |
| 记忆命中率 | 触发用户偏好规则的次数 / 可使用记忆的决策次数 |
| 端到端延迟 | 从候选机会生成到机器人完成表达的耗时 |

## 当前公开验证与研究边界

已完成：

- 自动化策略回放。
- 页面桌面端和移动端检查。
- Windows 启动包下载和脚本检查。
- 云端后端健康检查。
- 敏感信息扫描。

产品化阶段继续积累：

- 多用户长期使用数据。
- 用户主观体验评分。
- 长期记忆命中对提醒接受率的影响。
- 不同硬件形态下的反馈偏好差异。
