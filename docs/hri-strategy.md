# HRI 主动交互策略

## 决策原则

交互强度由四类因素共同决定：

1. 用户可用性：是否在会议、专注、阅读或离席。
2. 任务敏感度：当前任务是否容易被中断。
3. 状态置信度：关键信号是否真实可用。
4. 机器人可用性：是否在听、思考、说话或离线。

## 四级介入

| 等级 | 表现 | 适用场景 |
| --- | --- | --- |
| quiet | 只记录 | 信号不足、离席、硬件离线、冷却中 |
| subtle | 表情 + 轻动作 | 短暂返回、阅读思考、会议静默陪伴 |
| subtitle | 短字幕 + 表情/动作 | 消息处理、节奏分散、普通休息提示 |
| speak | 字幕 + 表情/动作 + 语音 | 高价值提醒、久别返回、疲劳上升 |

## 硬边界

- 会议中禁止普通主动语音。
- 机器人正在听/思考/说话时不抢占。
- 硬件离线时不生成“已执行”假象。
- 关键信号 unavailable 时不用窗口标题强行猜测。
- 主动语音、动作和字幕各自受冷却和每日预算限制。
