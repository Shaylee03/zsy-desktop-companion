# 隐私、风控与实现边界

## 隐私边界

- 默认不使用摄像头观察用户本人。
- 优先使用前台应用类别、窗口摘要、键鼠活跃、AFK、系统音频和设备占用等低隐私侵入信号。
- 不读取 WeChat / 飞书 / Teams 等消息正文。
- 不代替用户发送消息。
- 公开作品集不包含服务器凭据、API Key、原始桌面日志、用户记忆和 Hermes 会话。

## 主动行为风控

- 会议中禁止普通主动语音。
- 深度专注中普通提醒延后或降级。
- 信号不足时不做强判断。
- 硬件离线时只记录，不假装执行。
- 用户多次忽略或关闭某类提醒后，降低频率或降级强度。
- Agent 只生成话术和总结，不直接绕过规则触发高打扰行为。

## 实现边界

- 桌面状态识别是辅助判断，不等于完整理解用户意图。
- 低能量支持不是情绪诊断，只是工作节奏辅助。
- 分层记忆已支持写入和治理雏形，但记忆命中效果尚未形成稳定评估。
- 离席返回摘要和任务遗忘恢复仍在补齐工程闭环。
- 用户访谈和 3 天自测尚未形成真实结论前，不应在简历中写“显著降低误打扰率”或“用户接受率提升”。

## 推荐表达

可以说：

> 设计并构建低打扰主动式桌面 AI 机器人原型，建立状态感知、可打扰性判断、四级反馈和个性化记忆机制。

不建议说：

> 已商业化、已大规模验证、已显著降低误打扰率、能完整理解用户状态。
