# 实现说明

这份文档用于说明作品集页面中的主要说法分别由哪些材料支撑，方便外部读者快速判断项目当前做到什么程度。

## 快速查看路径

1. 首页：确认项目定位、产品闭环、核心场景、硬件联调和验证方式。
2. [系统架构](architecture.md)：查看本地感知端、后端策略、反馈层和记忆层的职责边界。
3. [主动交互策略](interaction-strategy.md) 与 [决策流](decision-flow.md)：查看不同状态下如何选择 `quiet / subtle / subtitle / speak`。
4. [记忆进化机制](memory-evolution.md)：查看上下文、偏好、节奏和反馈如何影响策略。
5. [验证记录](validation.md)：查看当前完成的规则路径、页面检查、本地链路和服务验证。
6. [小样本原型验证](small-sample-validation.md)：查看固定触发、状态降级、个性化记忆三组策略的指标口径。
7. [本地链路说明](download-and-run.md)：查看本地感知端如何连接后端和机器人。

## 页面说法与支撑材料

| 页面说法 | 支撑材料 | 支撑强度 |
| --- | --- | --- |
| 项目定位是主动式桌面 AI 陪伴机器人 | 首页、README、产品案例 | 高 |
| 面向研究生和办公人群 | 首页、产品案例、场景拆解 | 高 |
| 本地感知端使用非摄像头桌面信号 | 系统架构、本地链路说明、隐私边界 | 高 |
| 后端能根据状态选择反馈强度 | 主动交互策略、决策流、策略测试 | 高 |
| 低打扰是主动陪伴的交互边界 | 首页策略模块、交互策略文档 | 高 |
| 支持任务漂移、返回电脑、会议、专注等场景 | 产品案例、决策流、验证记录 | 中高 |
| 具备个性化记忆雏形 | 记忆机制文档、架构文档、验证记录 | 中高 |
| 已完成机器人硬件联调 | README、硬件模块说明、固件联调记录 | 中高 |
| 小样本指标有计算依据 | 小样本原型验证、验证脚本、README | 高 |

## 当前展示边界

- 页面展示的是个人产品原型，重点呈现产品定义、系统链路和验证方式。
- 硬件部分基于 ESP32-S3N16R8-EMOJI 开源硬件 / 固件框架进行适配和联调，底层硬件与固件遵循其原有来源和许可。
- 验证数据来自原型回放和小样本标注，用于说明策略方向，不等同于大规模用户实验。
- 完整运行需要本地感知端、云端后端和机器人硬件配对。
