# 证据地图

## 快速验证路径

1. 先看首页和 `docs/case-study.md`：确认项目命题是主动介入边界，不是普通提醒。
2. 再看 `docs/hri-strategy.md` 和 `docs/decision-flow.md`：确认四级反馈和硬边界。
3. 再看 `docs/architecture.md` 和 `docs/memory-evolution.md`：确认系统层级、记忆分类和 Agent 边界。
4. 如果想验证可传递性，看 `docs/download-and-run.md` 和 `downloads/zsy-desktop-agent-starter.zip`。
5. 最后看 `docs/validation.md` 和 `docs/research-and-self-test.md`：确认哪些指标已有口径，哪些仍待验证。

| 简历说法 | 当前证据 | 支撑强度 | 推荐表述 | 不应夸大为 |

## 公开仓库边界

不公开：

- 服务器凭据、API Key、`.env`。
- 原始桌面日志、真实对话日志。
- 用户记忆、Hermes 会话、硬件固件。

公开：

- 产品定位和取舍。
- 策略矩阵和决策流。
- 架构说明和字段口径。
- 脱敏后的指标口径和验证计划。
- Windows 本地 Agent starter 包、配置模板和启动/停止脚本。

## 下一步补证据清单

1. 会议、专注、离席、返回、任务切换各不少于 10 条标注样本。
2. 主动输出标注表：是否误打扰、是否接受、是否忽略、是否关闭。
3. 记忆命中日志：命中的记忆、影响的策略、用户是否纠正。
4. 硬件在线/离线回放：确认离线只记录，在线可成功下发字幕或动作。
