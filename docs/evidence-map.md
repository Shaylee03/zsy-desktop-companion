# 证据地图

这份文档用于回答面试官最常见的质疑：公开仓库里的材料，到底能支撑哪些简历说法，哪些还不能支撑。

## 快速验证路径

1. 先看首页首屏和“场景 demo”：确认项目不是普通聊天助手，而是围绕低打扰主动介入。
2. 再看 `docs/hri-strategy.md` 和 `docs/decision-flow.md`：确认策略不是“能说就说”，而是先判断可打扰性，再决定表达强度。
3. 再看 `docs/architecture.md`：确认感知、决策、表达和观测模块之间的责任边界。
4. 最后看 `docs/validation.md` 和 `docs/research-and-self-test.md`：确认哪些指标已有小样本口径，哪些还只是待验证计划。

## 简历表述支撑关系

| 简历可写内容 | 当前证据 | 可信程度 | 推荐写法 | 不建议写法 |
| --- | --- | --- | --- | --- |
| 低打扰主动交互原型 | 首页、README、产品案例 | 高 | 设计桌面 AI 伴侣机器人的低打扰主动交互原型 | 打造完整商业化机器人产品 |
| 非摄像头状态感知方案 | README、系统架构、隐私边界 | 中高 | 基于桌面信号辅助判断会议、专注、离席等状态 | 准确识别用户全部工作状态 |
| 四级反馈策略 | HRI 策略矩阵、首页策略表 | 高 | 将介入方式拆为 quiet/subtle/subtitle/speak 四级 | 实现全场景最优主动交互 |
| 降级与硬边界 | HRI 策略矩阵、决策流 | 高 | 设计会议、信号不足、硬件离线等降级策略 | 杜绝主动提醒误打扰 |
| Agent 辅助判断 | 系统架构、隐私与边界 | 中 | 搭建规则策略与 Agent 辅助判断框架 | 接入自主 Agent 完成主动决策闭环 |
| 分层记忆 | 系统架构、隐私与边界 | 中低 | 设计偏好、节奏和历史反馈的记忆原型 | 已验证记忆机制显著提升体验 |
| 延迟观测 | 指标与验证 | 中 | 建立 fast/deep 路由和延迟观测口径 | 已优化端到端响应性能 |
| 误打扰率、接受率 | 访谈与自测计划 | 低 | 规划误打扰率、接受率等体验指标 | 显著降低误打扰率 |

## 公开仓库边界

本仓库当前公开的是产品作品集和脱敏说明，不包含以下内容：

- 服务器凭据、API Key、原始桌面日志。
- 用户记忆、Hermes 会话、硬件固件。
- 能复现私有运行环境的完整服务端代码。

这些边界不影响作品集用于展示产品思考，但会影响“技术落地可信度”。如果要在面试中更强地证明落地，需要补充脱敏截图、运行录屏、日志片段和模块配置样例。
