Project Overview

把桌面状态、主动策略和具身反馈连接成闭环

当用户长期在电脑前学习和工作时，AI 能否理解当前状态，判断何时应该出现、何时应该保持安静，并逐渐适应用户的交互习惯？

Problem

现有 AI 能被动回答、解决问题，可以被动交互，
但很少有主动交互

长期学习与办公场景下，用户从不缺 AI 工具、被动应答与现成答案，真正稀缺的是懂分寸、有温度的智能陪伴，以及贴合当下状态的主动交互。AI 核心能力不在于给出解决方案，而在于精准识别用户此刻是专注工作、参会、临时离席还是切换任务，在恰当时机给出适配强度的反馈。

传统 AI 工具
  • 依赖用户主动调用
  • 缺少桌面状态理解
  • 反馈集中在文字或语音
  • 缺少连续的偏好依据
Moqi 的探索方向
  • 桌面信号形成状态摘要
  • 先判断主动机会是否成立
  • 按场景选择多模态反馈
  • 根据反馈更新后续策略
Moqi 重点解决两个问题什么时候应该主动介入？
介入时应该使用什么强度和组合的表达？

Interaction Strategy

主动不是更频繁地出现，而是在合适时使用合适表达

每次主动交互前，系统综合判断可打扰程度、事件重要度、状态置信度与近期反馈。

低置信度降级会议强制静默拒绝后冷却非必要不使用语音
Level 0

QUIET

保持安静

适用于会议、高专注、近期被拒绝、状态不明确或没有足够交互价值的场景。机器人不主动播报，也不执行明显动作，维持低存在感待机。

Level 1

VISUAL CUE

表情 + 短字幕

适用于紧急度较低、无需立即回应，但可以在自然停顿时注意到的场景。使用表情与短字幕组合，不把单一通道作为完整反馈。

Level 2

EMBODIED CUE

动作 + 表情 + 短字幕

适用于需要用户注意，但不适合用语音打断的场景。从已有动作库选择匹配意图的动作，并与表情和字幕组合。

Level 3

VOICE

简短语音 + 动作 + 表情 + 短字幕

适用于高优先级且用户可被打扰，或用户已进入交互状态的场景。语音保持简短，并按场景搭配表情或内置动作。

机器人动作来自第三方硬件的内置动作库。本项目负责根据交互场景选择动作，并与表情、短字幕和语音进行映射和联调，不涉及动作轨迹、动作幅度或底层舵机控制设计。

Key Iteration 01

单一表情或字幕不够明显，因此改为多模态组合

初始方案

为了控制打扰，最初尝试只通过机器人表情，或者只通过桌面字幕完成轻量提示。

实际发现

表情存在感较弱；字幕也容易与其他界面信息混在一起，用户不一定能够注意或理解机器人意图。

方案调整

没有直接增加语音频率：低强度使用表情与短字幕，需要注意时加入内置动作，信息重要且可打扰时才使用语音。

低打扰不等于反馈越弱越好。有效的低打扰反馈，需要在用户能够感知和理解的基础上控制表达强度。

Memory Mechanism

记忆不是保存聊天记录，而是让后续策略有依据

Moqi 将相对稳定的交互偏好、行为节奏和历史反馈转化为结构化记忆，减少重复犯错。

Preference

交互偏好

更偏好字幕还是语音、什么时间希望安静、某类提醒是否需要延后。

Rhythm

行为节奏

常见专注时间、会议集中时段，以及较稳定的离席和返回规律。

Feedback

反馈记忆

接受、忽略、延后、纠正和删除等反馈，为后续调整提供证据。

候选形成→总结合并→置信度评估→相关场景调用→反馈更新→衰减或删除
Example

专注场景下减少语音打断

写作或编程状态下多次关闭、忽略或延后语音提醒时，系统先形成低置信度候选，不把单次行为直接认定为长期偏好。

相似反馈重复出现，或用户明确确认“专注时不要主动说话”后，置信度才逐渐提高。再次进入相似场景时，普通提醒优先延后；确需提示时采用表情、内置动作与短字幕组合。

用户修改会立即更新记忆；长期没有新证据时逐渐降低权重。

可见、可纠正、可撤回

只有场景相关且置信度足够的记忆才影响策略。冲突或证据不足时降级、暂停调用或向用户确认。

Model Selection & Latency

模型选型

Moqi 的大模型负责理解桌面状态摘要、结合偏好生成简短反馈，并输出可供程序执行的结果。这类任务更强调速度、中文短表达、输出稳定性和调用成本。

01

状态理解

根据状态摘要理解用户可能处于专注、会议、离席或任务切换等状态。

02

表达生成

结合当前场景和相关偏好，生成适合机器人表达的简短中文反馈。

03

结构化输出

返回是否主动、反馈等级、表达内容和机器人能力指令，供程序执行。

会议静默、拒绝后冷却和隐私边界等确定性规则由程序控制，模型只在规则允许的范围内参与状态理解与内容生成。

Reflection

这个项目让我重新理解主动式机器人产品

主动的核心不是频率，而是时机；具身表达不是动作数量，而是用户能否感知并理解意图。

01

先学会保持安静

错误时间出现，比功能不足更影响体验。

02

表达需要组合

场景、表情、字幕、动作和语音共同形成意图。

03

用取舍建立核心体验

删除工具与降低延迟，比继续加功能更重要。

04

理解必须可控

个性化记忆需要可解释、可纠正和可删除。
