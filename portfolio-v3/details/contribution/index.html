Contribution & Boundaries

我完成了什么，复用了什么，没有做什么

这个项目的价值不在于自研机器人本体，而在于把桌面状态、主动策略、云端模型与已有机器人能力连接成可运行的产品闭环。

我完成的部分

工作覆盖产品定义、软件原型、策略机制、协议接入与三端联调。

产品与交互

产品定位、核心问题、目标场景与主动交互策略。

Windows 本地端

开源采集能力修改与接入、桌面状态摘要和产品化应用。

云端服务

后端、模型调用、主动与被动交互链路。

个性化记忆

候选、置信度、相关调用、反馈更新与衰减机制。

机器人接入

第三方协议接入及表情、动作、字幕、语音能力映射。

原型验证

三端联调、场景测试、模型选择、延迟优化和 Demo。

复用的外部基础

项目在现有软硬件能力上完成场景定义、策略设计、协议接入和多模态组合。

Software

开源桌面活动采集能力

在其基础上修改并接入 Moqi 本地端。

Model

第三方大语言模型 API

用于实时短回复与策略辅助，不涉及模型训练。

Hardware

第三方 ESP32 机器人本体

包括固件中的唤醒、收音、播放、表情与动作库。

当前未涉及

以下内容不应被描述为本项目产出。

硬件本体研发

机械结构、机器人本体、舵机和底层控制。

动作生成

动作轨迹、动作幅度、自主导航和抓取。

模型研发

大模型训练与微调。

商业化验证

量产设计和大规模用户实验。

一句话总结

我负责把产品定义、主动交互策略、Windows 与云端原型、结构化记忆、第三方机器人协议和多模态反馈连接起来，并通过真实原型发现问题、收敛范围和持续迭代。
