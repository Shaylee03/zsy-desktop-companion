系统与数据：Moqi 如何从桌面状态生成机器人反馈

本地端以最小必要信号形成状态摘要，云端完成规则判断、模型调用与记忆检索，机器人执行高层反馈指令。

当前数据来源

Moqi 的数据入口分为 Windows 本地端与机器人端：Windows 端理解桌面节奏和设备环境，机器人端承载语音与具身交互；云端不直接感知用户，而是对两端上传的脱敏信号进行融合、策略判断和记忆更新。

数据渠道数据类型具体信号主要用途处理边界
Windows 本地端桌面窗口状态当前应用、类别化窗口状态、停留时长、窗口切换判断任务类型、持续工作和任务切换真实标题仅用于本地判断；上云时转换为宽泛类别
Windows 本地端应用活动轨迹最近应用轨迹、近 30 分钟应用占比、活跃时长形成近期节奏和每日活动摘要本地 SQLite 独立记录；ActivityWatch 为可选增强源
Windows 本地端键鼠活跃度60 秒内按键、点击、滚动次数判断活跃、专注、低活动状态不记录具体按键、输入内容和鼠标位置
Windows 本地端AFK 与会话状态空闲时长、离席、返回、连续使用时间控制静默、返回陪伴和任务恢复只使用时间与状态信号
Windows 本地端麦克风与摄像头状态是否被应用调用、调用来源类型识别会议或通话状态不读取摄像头画面，不录制电脑麦克风内容
Windows 本地端系统音频状态是否播放音频、音频来源应用区分会议、媒体播放和普通工作不采集播放内容
Windows 本地端设备运行状态系统运行时间、进程数量、本地与云端连接状态运行诊断和故障判断完整进程与窗口列表不上云
Windows 本地端可选视觉理解当前活动窗口的任务类别、安全摘要和置信度补充元数据无法判断的桌面语义单独授权；仅限 HTTPS；图像只在内存中临时处理
Windows 本地端控制中心操作暂停感知、安静模式、隐私授权、记忆纠正和数据删除控制数据范围与交互偏好用户可随时暂停、纠正或删除
机器人端连接与设备状态WebSocket 在线状态、心跳、设备能力判断机器人是否可用待命在线不等于持续录音
机器人端交互状态待命、聆听、思考、说话、中断等状态协调语音会话和机器人反馈只记录交互阶段和必要时间信息
机器人端用户语音唤醒或聆听期间的麦克风音频、STT 文本理解用户请求并生成回应仅在明确进入聆听状态后处理
机器人端用户自然反馈语音中的接受、拒绝、纠正和偏好表达更新表达方式和场景策略先经过语义判断，再形成结构化反馈
云端处理层桌面状态摘要活动类别、注意状态、应用时长、AFK 和设备状态判断是否介入以及采用何种反馈层级接收脱敏摘要，不接收完整桌面原始数据
云端处理层视觉语义结果任务类别、安全摘要、敏感度、置信度辅助场景判断不保存截图或 Base64 图像
云端处理层对话上下文STT 文本、近期对话和当前任务线索保持交流连续性受长度、相关性和隐私规则限制
云端处理层策略事件静默原因、反馈层级、冷却、预算和机器人状态决定静默、表情、动作、字幕或语音模型不直接决定高打扰行为
云端处理层记忆与偏好场景偏好、节奏规律、用户反馈和置信度让 Moqi 逐渐适应用户支持确认、纠正、删除和时间降权

隐私原则

信息采集以完成状态判断为边界，长期记忆保存宽泛偏好与状态总结，而不是桌面原文。

01

最小化采集

只获取完成状态判断所需的信号。

02

本地优先

能够在本地整理的内容先生成状态摘要。

03

避免原文记忆

不保存或复述具体文件名、窗口标题与桌面内容。

04

用户可控

用户能够查看、纠正和删除形成的记忆。

运行状态

状态机限制主动输出的时机，并在反馈执行后进入冷却，避免连续打扰。

IDLE

默认待机，保持低存在感。

PASSIVE DIALOGUE

用户通过唤醒词发起对话。

PROACTIVE CANDIDATE

桌面状态满足潜在主动条件。

DO NOT DISTURB

会议、高专注或近期拒绝状态。

EXECUTING

执行表情、动作、字幕或语音反馈。

COOLDOWN

主动交互后降低频率，避免连续出现。

三端职责边界

Windows 本地端负责获取与整理信号；云端负责策略、模型与记忆；第三方机器人负责执行已有表情、动作、字幕和语音能力。云端不直接生成舵机轨迹。
