Model Selection & Latency

模型选型

实时桌面陪伴更强调响应速度、中文短表达、结构化输出稳定性和调用成本，不需要每次都进行复杂长链路推理。

模型在系统中负责什么

确定性规则先限定模型的参与范围，避免把会议静默、隐私和拒绝冷却交给模型自由判断。

State

状态理解

根据状态摘要理解用户可能处于专注、会议、离席或任务切换等状态。

Expression

表达生成

结合当前场景和相关偏好，生成适合机器人表达的简短中文反馈。

Schema

结构化输出

返回是否主动、反馈等级、表达内容和机器人能力指令，供程序执行。

规则优先于模型

会议静默、拒绝后冷却、隐私边界和高风险规则由程序控制；模型只在规则允许的范围内参与理解与生成。

场景化对比方式

围绕会议静默、专注状态、离席返回、久坐提醒和简短对话等典型任务，对比实时交互所需的关键能力。

响应延迟

请求发出到机器人开始反馈的时间，是否符合实时交互节奏。

中文短表达

能否在较短字数内自然表达，避免解释性过强的长段内容。

输出稳定性

能否稳定遵循字段、格式和字数要求。

场景理解

能否区分专注、会议、离席与短暂停顿。

调用成本

是否适合原型阶段反复调试和较高频率调用。

为什么最终选择 Flash

Flash 负责实时主动交互，记忆合并与冲突分析等复杂任务继续分层处理。

01

响应速度

等待过长会破坏对话与陪伴节奏，当前更需要尽快开始反馈。

02

任务不依赖长推理

实时链路主要完成状态理解、等级选择与简短内容生成。

03

中文短表达

配合字数约束，更适合机器人语音与桌面提示。

04

调用成本

适合原型阶段进行高频调用、提示词调整和三端联调。

Final Choice

当前实时链路：deepseek-v4-flash 快速非思考模式

快速通道迁移至显式 API 名称，复杂任务保留分层处理。

不同任务不必由同一个模型承担

实时任务、复杂任务与明确边界分别使用最合适的处理方式。

Realtime

实时链路

快速非思考模型负责状态理解、交互等级与简短回复。

Async

非实时链路

记忆总结合并与冲突分析可异步交给推理能力更强的模型。

Rules

硬规则层

会议静默、拒绝冷却、隐私与高风险边界不交给模型决定。

实时任务优先速度和稳定性，复杂任务优先推理能力，明确边界优先使用规则。

延迟迭代

记录原型迭代中典型场景的端到端响应体验。

V0 · 能力堆叠

部分交互超过 10 秒

接入搜索、时间和日历后，模型可能进入多步工具判断与调用。

V1 · 范围收敛

典型约 1.65 秒

删除非核心工具、限制搜索，并将回复约束为简短表达。

V2 · 链路优化

典型体验接近 1 秒级

采用更快的非思考模型并减少不必要请求步骤。

体验值来自典型场景记录，受网络与服务状态影响。

产品结论

这次优化不仅是模型或服务器调整，更是一次产品范围收敛：删除非核心能力，让实时陪伴的核心体验先成立。
