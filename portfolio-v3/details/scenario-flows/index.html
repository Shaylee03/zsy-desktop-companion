Scenario Flows

从触发信号到后续更新的完整场景流程

每个场景使用同一框架：触发信号 → 状态判断 → 策略选择 → 反馈执行 → 用户响应 → 后续更新。

PASS

离席返回

AFK 超过阈值→判断离席→暂停普通提醒→键鼠重新活跃→读取任务摘要→表情 + 动作 + 短欢迎语→根据反馈调整
PASS

会议静默

会议应用 / 音视频设备活跃→判断会议或通话→进入勿扰→禁止语音与明显动作→低存在感待机→状态结束后重评
IN PROGRESS

专注保护

持续输入 + 工作应用→判断高专注→普通提醒延后→低置信度静默→自然节点重判→必要时轻量反馈
NEXT

久坐提醒

持续使用达到候选阈值→检查专注与近期记录→判断提醒机会→动作 + 表情 + 字幕→接受 / 忽略 / 延后→调整时间与强度

流程中的共同边界

状态不明确时保持静默；会议与高专注优先于普通提醒；用户拒绝后进入冷却；反馈结束后才把响应作为后续策略证据。
