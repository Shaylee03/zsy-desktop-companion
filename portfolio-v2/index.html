主动式 AI 陪伴机器人

这是我独立搭建并持续迭代的可运行产品原型，目前重点验证主动交互、具身反馈与个性化记忆的核心闭环。

Project Overview

把桌面状态、主动策略和具身反馈连接成闭环

当用户长期在电脑前学习和工作时，AI 能否理解当前状态，判断何时应该出现、何时应该保持安静，并逐渐适应用户的交互习惯？

Problem

现有 AI 能回答问题，
但不知道什么时候该出现

长期学习和办公中，用户并不总是缺少答案。更关键的是，机器人能否理解当前是否正在专注、会议、离席或任务切换，并在合适的时机提供适当强度的反馈。

传统 AI 工具
  • 依赖用户主动调用
  • 缺少桌面状态理解
  • 反馈集中在文字或语音
  • 缺少连续的偏好依据
Moqi 的探索方向
  • 桌面信号形成状态摘要
  • 先判断主动机会是否成立
  • 按场景选择多模态反馈
  • 根据反馈更新后续策略
Moqi 重点解决两个问题什么时候应该主动介入？
介入时应该使用什么强度和组合的表达？

从本地桌面状态到机器人反馈的三端链路

本地端整理状态，云端判断主动机会与调用记忆，机器人及桌面端执行多模态反馈。

Interaction Strategy

主动不是更频繁地出现，而是在合适时使用合适表达

每次主动交互前，系统综合判断可打扰程度、事件重要度、状态置信度与近期反馈。

低置信度降级会议强制静默拒绝后冷却非必要不使用语音
Level 0

QUIET

保持安静

会议、高专注、近期拒绝或状态不明确时，不播报、不执行明显动作。

Level 1

VISUAL CUE

表情 + 短字幕

低紧急度信息在自然停顿中出现，避免单一表情或字幕难以被感知。

Level 2

EMBODIED CUE

内置动作 + 表情 + 短字幕

需要用户注意但不适合语音时，从已有动作库选择并组合反馈。

Level 3

VOICE

语音 + 具身反馈

仅在高优先级且可打扰，或用户已进入交互状态时使用简短语音。

Level 2 的动作轨迹与幅度来自硬件内置动作库；本项目负责选择、场景映射和多模态联调。

Key Iteration 01

单一表情或字幕不够明显，因此改为多模态组合

初始设想

只用表情或桌面字幕完成轻量提示，以降低打扰。

实际发现

表情存在感弱，字幕易与屏幕信息混在一起，用户不一定能理解机器人意图。

方案调整

低强度使用表情与短字幕；需要注意时加入动作；重要且可打扰时才使用语音。

低打扰不等于反馈越弱越好。有效反馈需要先让用户能够感知和理解，再控制表达强度。

Memory Mechanism

记忆不是保存聊天记录，而是让后续策略有依据

Moqi 将相对稳定的交互偏好、行为节奏和历史反馈转化为结构化记忆，减少重复犯错。

Preference

交互偏好

更偏好字幕还是语音、什么时间希望安静、某类提醒是否需要延后。

Rhythm

行为节奏

常见专注时间、会议集中时段，以及较稳定的离席和返回规律。

Feedback

反馈记忆

接受、忽略、延后、纠正和删除等反馈，为后续调整提供证据。

候选形成→总结合并→置信度评估→相关场景调用→反馈更新→衰减或删除
Example

“晚间优先字幕”如何形成

  1. 一次关闭语音，只形成低置信度候选。
  2. 相似场景中的重复行为与明确确认提升置信度。
  3. 进入相似场景时，优先使用视觉或具身提示。
  4. 用户修改立即生效；长期无证据则逐渐降权。
可见、可纠正、可撤回

只有场景相关且置信度足够的记忆才影响策略。冲突或证据不足时降级、暂停调用或向用户确认。

Key Iteration 02

从“功能越多越好”收敛为低延迟桌面陪伴

原型曾接入搜索、时间和日历工具，但部分交互达到十几秒，明显破坏即时陪伴感。

>10s初始工具调用版本
≈1.65s演示视频版本
≈1s后续核心链路

优化动作

  • 删除当前阶段非核心工具调用
  • 限制网页搜索与回复长度
  • 收敛系统提示词
  • 选择快速非思考模型
  • 调整云端部署与服务配置
这不只是技术提速，也是产品范围的重新收敛：删除非核心能力，让核心交互先成立。

当前使用 DeepSeek 的快速非思考模型；具体 API 型号将在核实后补充，不在页面中猜测。

Reflection

这个项目让我重新理解主动式机器人产品

主动的核心不是频率，而是时机；具身表达不是动作数量，而是用户能否感知并理解意图。

01

先学会保持安静

错误时间出现，比功能不足更影响体验。

02

表达需要组合

场景、表情、字幕、动作和语音共同形成意图。

03

用取舍建立核心体验

删除工具与降低延迟，比继续加功能更重要。

04

理解必须可控

个性化记忆需要可解释、可纠正和可删除。
