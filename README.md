# 低打扰主动式桌面 AI 机器人

这是一个面向学习、办公、远程协作和知识工作场景的个人独立项目。项目关注的核心问题不是“机器人能不能主动介入”，而是：

> AI 什么时候应该主动介入，什么时候应该延后，什么时候应该静默，什么时候才允许语音？

系统使用前台窗口、ActivityWatch、键鼠活跃、AFK、系统音频、会议设备状态和应用使用时长等非摄像头信号，辅助判断用户是否处在会议、深度专注、离席返回、任务切换、疲劳或空闲状态；再根据状态置信度、任务优先级、用户偏好和安全边界，在 `quiet / subtle / subtitle / speak` 四级反馈中选择最低可行打扰强度。

## 项目亮点

- 把主动式桌面助手的核心问题从“更频繁介入”转成“控制主动介入边界”。
- 覆盖会议勿扰、深度专注、离席返回、连续工作干预、任务恢复和低能量支持等桌面场景。
- 将机器人反馈拆成 `quiet / subtle / subtitle / speak` 四级，并在会议、专注、信号不足、硬件离线、冷却和预算约束下自动降级。
- 设计短期上下文、长期偏好、行为节奏和交互反馈四类记忆，用于持续调整介入时机和反馈强度。
- 保留本地采集端与连接链路说明，用于解释桌面状态如何进入后端策略层，再驱动机器人反馈。

## 产品闭环

```text
桌面信号
  -> 状态理解
  -> 可打扰性判断
  -> 反馈强度选择
  -> 用户响应
  -> 记忆更新
  -> 下一次更合适地介入
```

这条闭环对应页面中的核心展示：系统先判断用户是否处在会议、专注、离席返回、疲劳或空闲状态，再选择静默、轻动作、字幕或语音。用户接受、忽略、关闭、延后或纠正后，反馈会进入策略权重和记忆更新。

## 状态与反馈策略

| 用户状态 | 策略倾向 | 机器人表达 |
| --- | --- | --- |
| 会议中 | 禁止普通语音介入，低优先级事项静默记录 | `quiet` / `subtitle` |
| 深度专注 | 延后普通主动机会，必要时低压力字幕 | `quiet` / `subtitle` |
| 离席返回 | 先恢复上下文，不在返回瞬间语音打断 | `subtitle` |
| 连续工作 / 疲劳 | 结合输入强度、切换频率和历史反馈选择时机 | `subtle` / `subtitle` |
| 空闲且高置信 | 重要事项可使用更明确反馈 | `subtitle` / `speak` |

## 本地查看页面

本仓库是静态作品集页面，不需要构建流程。克隆后可直接打开 `index.html`，也可以用本地静态服务查看：

```bash
python -m http.server 8000
```

然后访问 `http://localhost:8000`。
