Personalized Memory

个性化记忆：如何形成、调用、校正和遗忘

Moqi 使用可解释的结构化记忆，根据反馈和时间变化更新策略；记忆始终受勿扰规则与用户控制约束。

机制案例

一次行为只形成候选，重复证据和明确反馈才会提高置信度。

Focus Preference

专注场景下减少语音打断

当用户在写作或编程状态下多次关闭、忽略或延后语音提醒时，系统先形成一条低置信度记忆候选。

如果相似反馈重复出现，或者用户明确确认“专注时不要主动说话”，偏好置信度才逐渐提高。之后进入相似场景时，Moqi 优先延后普通提醒；确实需要提示时，采用表情、内置动作与短字幕组合，而不是直接使用语音。

用户修改偏好时记忆立即更新；长期没有新证据时逐渐降低权重。

置信度如何确定

置信度由反馈强度、重复次数、跨场景一致性和时间新鲜度共同计算。

01

反馈强度

明确确认、纠正、修改或删除的权重高于隐式行为。

02

重复次数

同类行为多次出现，才逐步从候选转为稳定偏好。

03

跨场景一致性

证据在相似时间与任务状态下是否保持一致。

04

时间新鲜度

距离最近一次验证越久，记忆越需要重新确认。

信号强弱

单次没有回应不能直接解释为长期偏好；用户的明确操作应立即生效。

强信号

  • 用户明确确认
  • 用户明确纠正
  • 用户直接修改偏好
  • 用户删除记忆

弱信号

  • 接受或忽略
  • 延后
  • 关闭一次提醒
  • 单次没有回应

调用与更新规则

记忆必须与当前场景相关，且达到相应置信度，才能改变反馈方式。

高置信度

可以影响反馈方式，但仍受安全与勿扰规则约束。

中置信度

只做轻量调整，必要时向用户确认。

低置信度

保留为候选，不直接影响主动策略。

存在冲突

停止调用或降低强度，等待新证据。

时间衰减与用户控制

长期未验证的偏好逐渐降权；相反反馈让旧记忆进入冲突，而不是直接永久删除。

用户主动修改或删除时，立即生效

机器人对用户的理解可见、可纠正、可撤回。记忆管理界面和长周期衰减仍在持续完善。
